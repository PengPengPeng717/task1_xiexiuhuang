# 题 #1

## 主题: 24h 搭一个矿业新闻 + 政策 + 价格三源聚合管线

你将从下列 **3** 个数据源采集数据，每个源至少近 30 天 200 条（合计 ≥600 条），入向量库，提供一个 `/query` REST 接口可自然语言提问：

| # | 源           | URL                                                                                         | 难点                     |
|---|--------------|---------------------------------------------------------------------------------------------|--------------------------|
| 1 | 矿业新闻     | mining.com / S&P Global Mining (RSS 都给)                                                   | 全文需爬 + 结构化抽取    |
| 2 | 关键矿产政策 | 中国稀土集团官网（https://www.regcc.cn/） / 澳洲 DISR Critical Minerals Strategy（https://www.industry.gov.au/publications/critical-minerals-strategy-2023-2030） | 反爬中等 + HTML 结构不规整 |
| 3 | 价格         | LME 铜锌镍 + SHFE 锂 + 上海钢联铁矿石（[上海钢联-全球领先的大宗商品及相关产业数据服务商](https://www.mysteel.com/)） | 登录墙 / 接口频控        |

## 交付清单

- `pipeline/` — 采集 + 清洗 + 去重 + 入库
- `serve/` — FastAPI `/query` 接口，支持 “近 7 天澳洲锂出口政策有何变化？” 这类自然语言
- `eval/` — 20 条 ground truth Q&A，自动跑 `recall@5` + `answer faithfulness`
- `DATA_NOTES.md` — 描述 schema, 字段, 主键, 去重策略
